Server Name Indication（SNI）

SNI 是 TLS 协议中的一项关键扩展，用于在同一 IP 地址上安全托管多个域名， 是现代 HTTPS 基础设施的重要组成部分。

什么是 SNI

Server Name Indication（服务器名称指示）是 TLS 握手阶段的一个扩展字段。 客户端在建立加密连接之前，会将目标域名以明文形式发送给服务器， 使服务器能够选择正确的证书完成 TLS 握手。

为什么需要 SNI

虚拟主机

在同一 IP 地址上部署多个 HTTPS 站点时，SNI 允许服务器区分不同域名并返回对应证书。

IPv4 资源紧张

SNI 显著减少了对独立 IPv4 地址的依赖，是大规模部署 HTTPS 的现实解决方案。

现代浏览器支持

主流浏览器和操作系统均已支持 SNI，使其成为事实上的行业标准。

SNI 工作流程（简化）

ClientHello ├─ TLS Version ├─ Cipher Suites └─ Server Name: example.com ServerHello ├─ Selected Certificate (example.com) └─ TLS Handshake Continue

安全与隐私

由于传统 SNI 信息在 TLS 握手阶段是明文的，可能被用于流量分析。 为此，TLS 1.3 引入了 ESNI / ECH（Encrypted Client Hello）等机制， 用于加密服务器名称字段，进一步提升隐私保护能力。

应用场景

CDN 与云服务

在同一边缘节点上承载大量域名，实现高效的 HTTPS 分发。

反向代理

基于 SNI 将流量路由至不同后端服务或容器。

网络安全研究

分析 TLS 握手元数据，用于安全监测与协议演进研究。
