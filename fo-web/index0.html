让 HTTPS 迈入
多域名托管时代

SNI (Server Name Indication) 是 TLS 协议的扩展，允许在握手阶段指示目标主机名，从而实现单 IP 多证书部署。

深入了解原理

核心定义

在传统的 HTTPS 中，由于 SSL 握手发生在 HTTP 请求之前，服务器无法知道客户端想要访问哪个域名，因此只能返回默认证书。

"SNI 解决了这个问题：它在 TLS 握手的 Client Hello 阶段就将域名发送给服务器。"

这意味着您可以在同一个 IP 地址上为无限多个域名配置不同的 SSL 证书，极大地节省了 IPv4 资源。

// Client Hello with SNI

Handshake Type: Client Hello (1)

Version: TLS 1.2 (0x0303)

Extension: server_name

Server Name: sni0.com

通信时序对比

传统方式 (No SNI)

  • 1

    建立 TCP 连接

  • 2

    TLS 握手：服务器发送默认证书

  • !

    域名不匹配风险（如果不是默认站点）

SNI 方式

  • 1

    建立 TCP 连接

  • 2

    Client Hello：携带目标域名

  • 3

    服务器精准匹配并发送证书

节约 IP 资源

不再需要为每个 SSL 证书分配独立的 IPv4 地址，大幅降低运维成本。

虚拟主机增强

支持现代 CDN 和虚拟主机服务商在单台边缘节点上托管数以万计的站点。

安全性兼容

完全符合现代安全协议标准，并配合 ECH (加密客户端问候) 进一步保护用户隐私。
