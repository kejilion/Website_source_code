SNI 协议通过在加密握手初期明确目标域名，彻底革新了多站点 HTTPS 部署的逻辑。它是现代云计算与虚拟主机不可或缺的技术底层。

深度技术解析 RFC 文档

协议背景

在传统 SSL 架构下，服务器必须在解密流量前出示证书。然而对于共享 IP 的多域名环境，服务器面临着“不知道访客找谁”的困境。

SNI 的出现通过在 Client Hello 包中注入域名字段，赋予了服务器在加密建立前进行“精准分流”的能力。

TLS Handshake Sequence

PHASE 01 / TCP

基础连接

底层的 TCP 连接建立，确定物理链路可达性。

PHASE 02 / SNI

域名申明

客户端发送 Client Hello，在扩展字段中包含 SNI Header。

PHASE 03 / CERTS

证书匹配

服务器解析主机名，从映射表中提取匹配的 SSL 证书进行响应。

核心优势

SNI 不仅是技术进步，更是成本与效率的平衡点。

极致性能分发
大幅降低 IP 开销

云计算基石

现代负载均衡器（ALB/NLB）均基于 SNI 机制实现多租户证书隔离。

零延迟损耗

作为 TLS 扩展协议，它不会增加额外的往返时间（RTT）。
