Evolutionary Roadmap v3.0

INFRA EVOLUTION

从打破 IP 限制的枷锁，到构建全链路隐私加密，SNI 技术的发展见证了 Web 基础设施的蝶变过程。

ORIGIN / 2003

RFC 3546：协议雏形

在 SSL/TLS 握手早期，客户端无法告知服务器要访问的具体域名。RFC 3546 首次提出了在握手中加入 server_name 扩展的构想，为虚拟主机时代的 HTTPS 奠定了基石。

Specification Details
Extension Type 0x0000
Status Proposed
Impact: IPv4 conservation achieved.
EXPANSION / 2011 - 2015

虚拟主机的黄金时代

随着 RFC 6066 的标准化，SNI 在主流浏览器和服务器（Nginx/Apache）中得到全面支持。这直接催生了云服务商的弹性负载均衡（ALB/NLB），让单 IP 托管数万个 HTTPS 站点成为低成本的现实。

VULNERABILITY / 2018

明文泄露风险

虽然 SNI 解决了路由问题，但它的明文传输特性成为了隐私的“最后一公里”漏洞。中间人（MITM）或 ISP 可以通过嗅探 Client Hello 包中的 SNI 字段，轻松得知用户正在访问哪个网站。

Security Alert
Packet Sniffed:
TLS Header [SNI: confidential.com]
Status: Exposed

ECH 工作流

使用服务商公钥加密内层 Client Hello
中间人仅能看见“假外壳”域名
实现真正的端到端域名加密
FUTURE / 2024 - BEYOND

ECH：全链路隐身

从 ESNI 进化到 ECH (Encrypted Client Hello)，这是 TLS 1.3 后的最重磅补丁。它将整个握手负载进行加密，使得外部观察者完全无法窥探访客的真实目的。

技术总结

SNI 的演进史就是一部互联网在“效率”与“安全”之间不断平衡的历史。
从最初为了解决 IPv4 枯竭的无奈之举，到如今成为保护数字人权的关键一环，
SNI0.COM 邀您共同见证更安全的 Web 未来。
